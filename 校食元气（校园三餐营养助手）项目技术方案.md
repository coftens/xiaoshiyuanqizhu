## 一、技术选型总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **移动端** | **Flutter 3.x (Dart)** | 一套代码编译 Android + iOS，Skia 自绘引擎，UI 表现力极强 |
| **状态管理** | Riverpod 2.x | 声明式、可测试、支持代码生成 |
| **路由** | GoRouter | 声明式路由，支持深度链接和路由守卫 |
| **后端** | **Spring Boot 3.x (Java 17)** | 企业级框架，生态成熟 |
| **ORM** | MyBatis-Plus | 简化 CRUD，支持分页和动态 SQL |
| **鉴权** | JWT + Spring Security | 无状态令牌认证 |
| **数据库** | MySQL 8.0 | 结构化业务数据存储 |
| **缓存** | Redis 7.x | 推荐结果缓存、会话、排行榜 |
| **AI 模块** | DeepSeek / 通义千问 API | 大模型文本推荐 + 多模态识图 |
| **对象存储** | 阿里云 OSS / MinIO | 用户头像、食物照片 |
| **天气服务** | 和风天气 API（免费版） | 季节性/天气饮食调整 |
| **推送** | 极光推送 / Firebase Cloud Messaging | 餐前提醒、步数督促 |
| **部署** | Docker + Nginx | 容器化 + 反向代理 |
| **版本控制** | Git + GitHub/Gitee | 代码管理与协作 |

## 二、为什么选 Flutter？

| 对比 | Flutter | React Native | UniApp |
|------|---------|-------------|--------|
| **UI 自由度** | ⭐⭐⭐⭐⭐ 自绘引擎，像素级控制 | ⭐⭐⭐ 依赖原生组件 | ⭐⭐⭐ WebView 渲染 |
| **动画能力** | ⭐⭐⭐⭐⭐ 60fps 流畅 | ⭐⭐⭐ 桥接开销 | ⭐⭐ 性能受限 |
| **双端一致性** | ⭐⭐⭐⭐⭐ 完全一致 | ⭐⭐⭐ 平台差异 | ⭐⭐⭐ 兼容问题 |
| **性能** | ⭐⭐⭐⭐⭐ AOT 编译 | ⭐⭐⭐⭐ JS 桥接 | ⭐⭐⭐ WebView |
| **Widget 支持** | ✅ home_widget 插件 | ⚠️ 需原生模块 | ❌ 不支持 |

> **结论**：你对 UI 要求很高，加上需要桌面 Widget、自定义动画（课表矩阵、餐盘可视化、BMI 色彩条），Flutter 的 Skia 自绘引擎是最优解。

## 四、核心模块技术设计

### 4.1 用户模块

| 功能 | 技术方案 |
|------|---------|
| 注册/登录 | 手机号 + 短信验证码（阿里云短信 / 腾讯云短信） |
| JWT 鉴权 | Access Token (2h) + Refresh Token (7d)，双 Token 机制 |
| 极速建档 | 分步表单 → 一次提交到后端，后端即时计算 BMI/BMR/TDEE |
| 智能默认体重 | 前端在选择身高后即时计算：`默认体重 = 22 × (身高m)²` |

### 4.2 课表模块

| 功能 | 技术方案 |
|------|---------|
| 作息时间设定 | 存储上午/下午/晚课的开始结束时间 + 节数 |
| 动态网格生成 | 根据节数自动生成 5（周一~五）× N（总节数）矩阵 |
| 三色状态 | 每个格子存储枚举值：`NONE / CULTURE / SPORTS / SCIENCE` |
| 课程名称 | 长按编辑，存储到 `schedule` 表的 `course_name` 字段 |
| 周复制 | API: `POST /schedule/copy-last-week`，复制上周记录并更新 week_start_date |
| 能量建模 | 按日统计各类型课时数 × 对应消耗系数，合成 PAL |
| 实时课程感知 | 前端根据 `当前时间 + 课表数据` 本地计算当前所在课程 |

### 4.3 营养算法引擎

```
输入：用户体征 + 课表 + 今日状态补丁 + 天气 + 步数
  ↓
Step 1：BMR 计算（Mifflin-St Jeor 公式）
  男：10×体重(kg) + 6.25×身高(cm) - 5×年龄 + 5
  女：10×体重(kg) + 6.25×身高(cm) - 5×年龄 - 161
  ↓
Step 2：PAL 动态活动系数合成
  PAL = 基础系数(1.2)
      + 课表加成（Σ 各节课消耗系数 × 权重）
      + 步数加成（步数/10000 × 0.2）
      + 体育课加成（有体育课当天额外 +0.15）
  ↓
Step 3：TDEE = BMR × PAL
  ↓
Step 4：目标热量 = TDEE + 目标调整
  减脂：-300~500 kcal | 增肌：+200~300 kcal | 维持：±0
  ↓
Step 5：状态补丁修正
  生理期：+100 kcal，铁需求 ×1.5
  考试周：碳水 +10%（供能大脑）
  运动日：蛋白 +20%，碳水 +15%
  天气修正：冬天 +5~10%，夏天 -5%
  ↓
Step 6：宏量素分配
  蛋白质：体重 × 1.2~2.0g（目标决定系数）
  脂肪：总热量 × 20~30%
  碳水：剩余热量 ÷ 4
  ↓
Step 7：三餐拆分
  默认：早30% : 午40% : 晚30%
  早八满课日：早35% : 午35% : 晚30%
  晚课日：午35% : 晚35% : 晚加餐 5%
  ↓
Step 8：饮水量计算
  基础：体重(kg) × 30ml
  修正：体育课 +500ml | 高温天 +300ml
  ↓
输出 → AI 推荐模块（生成具体食物建议）
```

### 4.4 AI 推荐模块（大模型集成）

#### 4.4.1 文字推荐（核心）

```java
// Prompt 模板结构
public class MealPromptBuilder {
    String systemRole;     // AI 营养顾问角色设定
    UserContext context;   // 体征 + BMI + 目标
    ScheduleContext schedule; // 今日课表 + 当前状态
    NutritionTarget target;  // 本餐热量 + 三大宏量素目标
    DietRestriction restriction; // 过敏源 + 禁忌 + 偏好
    StatusPatch patch;     // 生理期 / 考试周 / 天气
    String outputFormat;   // JSON 格式要求
}
```

**技术要点：**
- 后端 `AiRecommendService` 统一封装，支持多模型切换（DeepSeek / 通义千问）
- **SSE 流式响应**：`@GetMapping(produces = MediaType.TEXT_EVENT_STREAM_VALUE)`
- Flutter 前端用 Dio + StreamTransformer 逐字渲染
- Redis 缓存：相同 `prompt_hash` 直接返回缓存，有效期 6 小时
- **Fallback 机制**：API 不可用时降级为本地规则引擎（预置 50+ 食物搭配模板）

#### 4.4.2 AI 气泡生成

| 生成方式 | 说明 |
|---------|------|
| **本地规则优先** | 基于时间 + 课表 + 状态的模板匹配（300+ 条文案库），不消耗 API |
| **大模型增强** | 每天首次打开 App 时调用一次 API 生成 3 条个性化气泡，本地缓存 |
| **混合策略** | 联网用大模型，离线用本地模板，用户无感知 |

#### 4.4.3 拍照识食（多模态）

```
拍照 → 压缩至 < 1MB → 上传 OSS 获取 URL
  → 调用通义千问 VL / DeepSeek-VL API
  → 返回 JSON: { foods: [{name, weight_g, calories, protein, carb, fat}] }
  → 前端展示识别结果，用户确认/修正
  → 写入 meal_record 表
```

### 4.5 反馈闭环 & 智能纠偏

| 反馈类型 | 后端逻辑 |
|---------|---------|
| 按建议吃了 | 正向记录 +1，优化推荐权重 |
| 吃多了 | 下一餐目标热量 × 0.85，或推荐补偿运动（步数目标 +2000） |
| 没吃饱 | 下一餐目标热量 × 1.10，优化碳水和蛋白配比 |

    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    food_name       VARCHAR(50) NOT NULL,
    category        VARCHAR(20),    -- PROTEIN / CARB / VEGETABLE / FAT
    calories_per_100g INT,
    protein_per_100g  DECIMAL(4,1),
    carb_per_100g     DECIMAL(4,1),
    fat_per_100g      DECIMAL(4,1),
    substitute_group  INT           -- 同组内可互相替换
);
