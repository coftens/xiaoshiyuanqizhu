## 二、逐页详细设计

---

### 【Page 1】启动屏

**目的**：品牌第一印象，传递产品调性——温暖、专业、懂大学生。

| 要素 | 说明 |
|------|------|
| **背景** | 全屏高品质图片，风格：校园晨光 / 食堂暖色调 / 抽象渐变（可轮换） |
| **文案** | 中央大字，随机展示一句激励语（见下方文案库） |
| **Logo** | 底部居中，"校食元气" 品牌标识 |
| **动效** | 文字淡入（fade in）+ 轻微上浮，背景微缩放呼吸感 |
| **时长** | 停留 2.5 秒后自动渐隐过渡到下一页 |

#### 激励语文案库（随机展示）

```
• "早八不饿，满课不累，三餐刚好。"
• "吃对每一餐，元气满满过四年。"
• "你的课表，我来配餐。"
• "别让早八打败你的胃。"
• "科学吃饭，不卷身材卷健康。"
• "今天也要好好吃饭呀。"
```

### 【Page 2】引导页：用户信息设置

**整体风格**：卡片式分步引导，每步一屏，左右滑动或点击"下一步"推进。顶部有**进度条**（如 ●●●○○○），底部有"下一步"按钮。背景使用品牌主色渐变。

#### 2a. 性别选择

| 要素 | 说明 |
|------|------|
| **标题** | "先认识一下你 👋" |
| **交互** | 两个大圆形卡片：♂ 男生 / ♀ 女生 |
| **动效** | 选中时卡片放大 + 边框高亮 + 轻微弹跳（spring animation） |
| **设计细节** | 卡片内可配上简约插画（男生/女生剪影） |

---

#### 2b. 身高 & 年龄

| 要素 | 说明 |
|------|------|
| **标题** | "告诉我你的身高和年龄" |
| **身高输入** | 数字滚轮选择器，范围 140cm–210cm，默认 170cm（男）/ 160cm（女） |
| **年龄输入** | 数字滚轮选择器，范围 16–30岁，默认 20岁 |
| **滚轮样式** | 简约大气风格：选中项放大加粗 + 上下项透明度递减，无多余边框装饰 |

---

#### 2c. 体重选择（核心亮点交互）

| 要素 | 说明 |
|------|------|
| **标题** | "你的体重是？" |
| **交互** | 大号滚轮选择器，范围 35kg–150kg，步长 0.5kg |
| **⭐ 智能默认值** | 根据 2a/2b 已选的性别+身高+年龄，用 BMI=22（中国标准体重 BMI）反算默认体重 |
| **计算公式** | `默认体重 = 22 × (身高m)²`，例：175cm → 22 × 1.75² = **67.4kg** |
| **副文案** | 滚轮下方小字："基于您的身高，标准体重约为 67.4kg" |
| **滚轮样式** | 居中大字体展示当前值 + 单位 "kg"，上下渐隐，整体极简 |

---

#### 2d. BMI 结果 & 健康分析（小惊喜页）

用户确认体重后，**过渡动画**展示计算过程（数字跳动效果），然后呈现结果卡片：

| 要素 | 说明 |
|------|------|
| **BMI 数值** | 居中大号字体，如 "22.9" |
| **体型标签** | BMI 色彩条 + 指针定位，标签：偏瘦 / 正常 / 超重 / 肥胖 |
| **健康寄语** | 根据 BMI 区间 + 性别生成个性化文案（见下方） |
| **设计** | 卡片式布局，毛玻璃背景，渐变色彩条 |

##### BMI 区间与文案逻辑

| BMI 范围 | 标签 | 文案示例 |
|----------|------|---------|
| < 18.5 | 偏瘦 | "你有点偏瘦哦~不过没关系，我会帮你科学增重，吃出好身材！目标是在保证营养均衡的前提下，适当增加优质蛋白和碳水摄入。" |
| 18.5–23.9 | 正常 | "身材管理得不错！你的 BMI 处于健康区间，接下来我帮你维持好状态，让每一餐都吃得聪明、吃得舒服。✌️" |
| 24.0–27.9 | 超重 | "体重稍微超标了一点点，但完全可控！我会结合你的课表，帮你用最轻松的方式，把体重慢慢调回来。💪" |
| ≥ 28.0 | 肥胖 | "我理解你的困扰，但这不是一个人的战斗。接下来我会陪你一步步调整饮食节奏，不节食、不痛苦，科学地变健康。🤝" |

##### 附加健康估算（基于体征计算，增加专业感）

```
📊 你的身体快照：
• 基础代谢率 (BMR)：约 1,587 kcal/天
• 体脂率估算：约 18%（正常范围）
• 每日饮水建议：约 2,100ml（约 8 杯水）
• 蛋白质需求：约 84g/天（约 3 个掌心大小的肉类）
```

> 注：体脂率使用 BMI 估算公式（非精确值），标注"仅供参考"。

### 【Page 3】作息时间设置

**目的**：获取用户学校的上下课时间，用于自动生成课表网格。

| 要素 | 说明 |
|------|------|
| **标题** | "设置你的学校作息 🏫" |
| **副标题** | "帮我了解你的上课时间，我来帮你安排三餐" |

#### 设置项

| 时段 | 设置内容 | 示例 | 必选 |
|------|---------|------|------|
| **上午** | 上课时间 / 下课时间 / 课节数 | 08:00–12:00 / 4节 | ✅ 必选 |
| **下午** | 上课时间 / 下课时间 / 课节数 | 14:00–17:30 / 3节 | ✅ 必选 |
| **晚课** | 上课时间 / 下课时间 / 课节数 | 19:00–21:00 / 2节 | ⭕ 非必选 |

#### 交互细节

- 时间选择使用**时间滚轮**（时:分格式），与之前体重滚轮风格一致
- 课节数使用**数字选择器**（1–6节）
- 晚课区域有开关（Switch），默认关闭，打开后展开设置项
- 底部"生成课表"按钮，点击后自动计算出网格结构

#### 三色课程类型

| 颜色 | 类型 | 能量消耗系数 | 含义 |
|------|------|------------|------|
| 🟩 绿色 | 文化课 | 低消耗（×1.0） | 听讲为主，脑力消耗低（如思政、英语、通识课） |
| 🟨 黄色 | 理科课 | 中消耗（×1.2） | 高强度用脑（如高数、物理、编程、实验课） |
| 🟥 红色 | 体育课 | 高消耗（×1.8） | 高体力消耗（体育课、体测训练） |
| ⬜ 空白 | 无课 | 基础（×0.8） | 自由时间，可安排运动或休息 |

#### 点击交互逻辑

```
空白 ──点击──▶ 🟩 绿色（文化课）
  ▲                │
  │              点击
  │                ▼
点击            🟥 红色（体育课）
  │                │
  │              点击
  │                ▼
  └──────────── 🟨 黄色（理科课）
```

#### 动效要求

- 每次点击有**颜色渐变过渡**（200ms），不是生硬切换
- 方块选中瞬间有**轻微缩放弹跳**（scale 0.95 → 1.0）
- 长按可**快速清除**该方块（回到空白）
- 每行（同一时段）之间有**微妙的分割线**

#### 7.2 AI 气泡评语（核心体验）

**不是对话界面，而是信息气泡形式**，像一个贴心的朋友在首页给你"留便签"。

| 触发时机 | AI 气泡内容示例 |
|---------|----------------|
| **早晨打开 App** | "早上好~ 今天上午满课，早餐记得吃饱，推荐全麦面包+水煮蛋+牛奶 ☀️" |
| **上课中** | "第3节高数中... 好好听课！中午奖励自己一份好吃的 💪" |
| **午餐前** | "快下课啦！午餐推荐已出炉，今天的蛋白质还差 35g 💡" |
| **下午空闲** | "下午没课，适合散步 20 分钟消化午餐~ 目前步数 2,800，加油！🚶" |
| **晚上** | "今天步数达标了！晚餐可以稍微放松，但别忘记蔬菜哦 🥦" |
| **考试周** | "考试周辛苦了，给大脑补补能量：坚果+香蕉+黑巧克力 🧠" |
| **生理期** | "这几天注意保暖，推荐温热饮食，冰的先放几天吧~ ❤️" |

##### 气泡设计规范

- 外观：圆角卡片 + 左上角 🤖 AI 头像小图标
- 背景：品牌色半透明毛玻璃
- 动效：新气泡从右侧滑入，旧气泡淡出
- 频率：**不超过 3 条/天**，避免打扰
- 可关闭：左滑隐藏当条气泡

## 四、我补充的功能建议

以下是结合你的设计文档和我的经验，额外建议的功能点：

### 4.1 🍎 "现在还能吃什么"速查

> 来源：你原始文档提到的"现在还可以吃什么"。

| 要素 | 说明 |
|------|------|
| **入口** | 首页浮动按钮 "🍎 现在吃啥？" |
| **逻辑** | 根据当前时间、已吃餐次、剩余热量额度，AI 即时推荐 |
| **场景一** | 两餐之间饿了 → "你还有 200kcal 的零食额度，推荐：1个苹果 或 10颗杏仁" |
| **场景二** | 晚餐后嘴馋 → "今日热量已达标，如果实在想吃，选无糖酸奶最安全~" |
| **场景三** | 运动后 → "刚运动完，建议补充：1杯牛奶 + 1根香蕉，帮助肌肉恢复" |

### 4.2 🔔 智能推送提醒

| 时机 | 推送内容 |
|------|---------|
| 早餐前 30 分钟 | "早上好~ 今天的早餐推荐已出炉，点击查看 🌅" |
| 午餐前 30 分钟 | "快下课了吧？午餐吃这几样，下午不犯困 ☀️" |
| 下午空闲时段 | "今天步数 3,200，离目标还差 4,800步，散步去？🚶" |
| 睡前 | "今日饮食小结：蛋白质达标 ✅ 蔬菜略少 😅 明天多吃点绿叶菜~" |

### 4.3 🏅 成就与激励系统

| 成就 | 解锁条件 |
|------|---------|
| 🌱 健康萌新 | 完成首次建档 |
| 🔥 连续打卡 3 天 | 连续 3 天进行餐后反馈 |
| 💪 一周达人 | 一周内 80% 餐次按建议执行 |
| 🏃 步数之星 | 单日步数达到 10,000 |
| 🧠 高压战士 | 考试周坚持健康饮食满 5 天 |

> 成就解锁时全屏撒花动画 + 轻震动反馈，制造惊喜感。

### 4.4 🌙 夜猫子模式

大学生熬夜是常态，如果检测到用户经常在 23:00 后活跃：

- 自动建议"宵夜方案"：低热量 + 助眠食物（牛奶、香蕉、坚果）
- 提示："熬夜伤身，但如果一定要熬，至少让胃不受苦 🌙"
- 次日早餐自动延后推荐时间，适配晚起场景

### 4.5 👥 舍友/好友模式（社交裂变）

- 邀请舍友加入，对比本周饮食执行率
- "舍友 PK"排行榜，增添趣味
- 有助于比赛展示"社交传播"能力，评委加分项

### 4.6 📸 拍照识食（比赛超级加分项）

这是近年来健康App的热门功能，**对比赛评委来说是一个很大的技术亮点**。

| 要素 | 说明 |
|------|------|
| **入口** | 首页或饮食页的相机按钮 📸 |
| **流程** | 拍食物照片 → 大模型识别食物种类 → 估算热量和营养素 → 自动记录到今日饮食 |
| **技术** | 将图片发送到后端，通过大模型的多模态能力（如通义千问VL / GPT-4V）识别 |
| **交互** | 拍照后 2 秒内出结果，展示"识别到：鸡腿饭 + 紫菜蛋花汤，约 750kcal" |
| **用户修正** | 识别结果可手动调整，系统学习用户纠正数据 |
| **价值** | 解决"不知道自己吃了多少卡路里"的问题，但不需要用户手动搜索和记录 |

> 💡 这个功能可以作为 V2 版本的亮点，MVP阶段可以先用文字版推荐。

### 4.7 💧 饮水追踪与提醒

大学生普遍饮水不足，这是一个简单但有价值的功能。

| 要素 | 说明 |
|------|------|
| **入口** | 首页今日状态区域，显示水杯图标 + 进度 |
| **目标** | 根据体重自动计算（体重kg × 30ml），如 70kg → 2100ml |
| **记录方式** | 点击 "+1杯"（250ml）快速记录，也可自定义量 |
| **提醒** | 每 2 小时推送一次饮水提醒（上课时段自动静音） |
| **与饮食联动** | 运动后 / 体育课后自动增加饮水建议量 |
| **可视化** | 水杯从空到满的填充动画，满了撒水花特效 🎉 |

### 4.8 🌤️ 季节性 & 天气饮食调整

| 场景 | 自动调整 |
|------|---------|
| **冬天 / 降温天** | 推荐热汤、炖菜，增加热量 5-10% |
| **夏天 / 高温天** | 推荐清淡、凉拌类，增加饮水建议量 |
| **雨天** | "下雨天窝在宿舍？记得喝杯热水暖暖身~" |
| **换季期** | 推送免疫增强食材（维C水果、蜂蜜柠檬水） |
| **技术实现** | 接入免费天气 API（如和风天气），根据定位获取实时天气 |

### 4.10 🔄 食物替换建议

## 五、设计风格指南

| 维度 | 规范 |
|------|------|
| **主色调** | 温暖绿（#4CAF50 系）+ 暖橙点缀（#FF9800 系），传递健康活力 |
| **暗色模式** | 深灰底（#1A1A2E）+ 渐变色卡片，夜间护眼 |
| **字体** | 中文：思源黑体 / PingFang SC；数字：DIN Pro / SF Pro |
| **圆角** | 卡片统一 16px 圆角，按钮 24px 全圆角 |
| **动效原则** | 每个交互都有微动效，但不过度：点击反馈 < 150ms，页面转场 300ms |
| **图标风格** | 线性图标 + 品牌色填充，统一 24px 尺寸 |
| **留白** | 大量留白，信息密度适中，不拥挤 |
